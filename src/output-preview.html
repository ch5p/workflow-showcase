여기에 XML 또는 영상 파일을 놓으세요

1280 Primary Timeline — 24fps Video + 60fps UI

기본 흐름

프리미어: 시퀀스 선택 → 파일 → 내보내기 → Final Cut Pro XML → 나온 .xml 파일을 이 창에 드래그&드랍. 현재 보이는 최상단 활성 클립을 기준으로 PRIMARY 타임라인이 자동 생성된다. 토탈 길이는 XML의 시퀀스 길이를 그대로 따른다 (10초든 30초든 자동).

단축키

Space 재생/일시정지 · R 리셋 · D 디버그 · H 도움말

구간만 사용하기

파일을 메모장으로 열고 CONFIG.range를 지정: range: { in: "00:00:05:00", out: "00:00:15:00" } (타임코드 또는 초 숫자). 해당 구간만 잘라 0초부터 재정렬된다. 또는 CONFIG.useWorkArea = true로 두면 프리미어 작업 영역(시퀀스 메뉴 → 작업 영역 막대 표시) 구간을 자동으로 읽는다. 수동 range가 있으면 그게 우선.

녹화 (권장: OBS 브라우저 소스)

OBS에서 소스 추가 → 브라우저 → 로컬 파일 체크 후 이 html 선택 → 너비 1280 / 높이 1080. Space 입력은 소스 우클릭 → 상호 작용 창에서. 일반 화면 녹화 시엔 주소 뒤에 ?scale=1.

실제 영상 싱크

완성 영상을 화면에 드롭하면 영상 시간이 타임라인의 기준 시계가 된다. XML 길이와 영상 길이가 1프레임 이상 다르면 영상 왼쪽 위에 DURATION Δ가 표시된다. D를 누르면 video/virtual clock과 drift 값을 확인할 수 있다.

Focus Timeline

상세 타임라인은 기본 7초 단위로 전환되고, 하단 FULL 바는 전체 영상 위치를 계속 표시한다. XML에서 비활성화한 클립은 PRIMARY 계산에서 제외된다. 페이지 길이는 CONFIG.timelinePaging.pageSeconds에서 조정한다.

Mixed FPS Preview

영상은 XML의 원본 프레임 속도로 재생하고, UI는 CONFIG.uiFps 기준으로 갱신한다. 둘은 프레임 번호가 아니라 동일한 재생 시간으로 연결되므로 영상·UI·오디오 길이는 변하지 않는다.

스타일 조정

CONFIG.panelHeight로 패널 높이, CONFIG.showTimecode로 타임코드 표시 여부를 바꿀 수 있다.

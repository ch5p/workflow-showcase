# AI로 커스터마이징하기 — 코드 몰라도 LLM한테 시키기

[`English →`](./CUSTOMIZING_WITH_AI.md) · `한국어`

이 문서는 **코드를 직접 보지 않고**, LLM(코덱스·클로드 등)에게 "이거 이렇게 바꿔줘"라고 시켜서 이 앱을 자기 취향대로 바꾸는 사람을 위한 안내서입니다. 이 프로젝트는 애초에 그렇게 쓰라고 만들었습니다.

담고 있는 것:

1. LLM한테 시킬 때 먼저 붙일 **기본 주문**(복붙용)
2. **자유롭게 바꿔도 되는 것 / 조심할 것** 신호등
3. **"이걸 바꾸고 싶으면 이렇게 말하세요"** 인텐트 표
4. 바꾼 뒤 **확인하는 법**과 **되돌리는 법**

> 이 문서 자체는 마음대로 고쳐도 됩니다. 앱 동작과 무관한 순수 안내문입니다.

---

## 0. LLM한테 시킬 때 먼저 붙일 기본 주문 (복붙)

무언가 바꿔달라고 할 때, 요청 앞이나 뒤에 이걸 그대로 붙이세요. 그러면 LLM이 사고 칠 확률이 크게 줄어듭니다.

```
이 저장소를 고치기 전에 AGENTS.md, docs/PROJECT_MAP.md의 계약,
CUSTOMIZING.md의 Safe/Stable 구분을 먼저 읽어줘.
내 요청이 Red Zone이나 Stable core에 걸리면 바로 고치지 말고 먼저 알려줘.
runtime 오류라면 코드를 고치기 전에 docs/TROUBLESHOOTING.md를 따라 진단해줘.
기존 미커밋 변경은 보존하고 이번 요청에서 바꿀 파일과 이유를 먼저 알려줘.
다 고친 다음엔 npm.cmd run check, 그다음 npm.cmd run smoke 를 실행해줘.
Export 계약을 바꿨다면 npm.cmd run smoke:export 도 실행해줘.
화면으로 봐야 하는 변경이면 내가 어디를 눌러 확인하면 되는지 알려줘.
```

이 한 덩어리가 하는 일: LLM이 (1) 위험 구역을 먼저 파악하고 → (2) 기존 작업을 보존하고 → (3) 고친 뒤 자동 검사를 돌리고 → (4) 당신이 눈으로 확인할 방법까지 남기게 만듭니다.

## 1. 신호등 — 바꿔도 되는 것 / 조심할 것

| 구역 | 무엇 | LLM 혼자 끝낼 수 있나 |
|---|---|---|
| 🟢 **자유** | 화면 **문구·도움말**, **버튼 이름**, 공통 **색상·디자인 토큰**(`tokens.css`) | 예. 고치고 자동 검사까지 가능 |
| 🟡 **물어보고** | **출력 해상도·화면비**, classic **배치**(`classic.css`), 창 크기, 자막(콜아웃) 크기·색 | 가능하지만 "무엇을 바꾸는지" 보고를 먼저 받고 진행. 화면비/해상도는 [`../CUSTOMIZING.ko.md`](../CUSTOMIZING.ko.md)의 포크 가이드가 정식 경로 |
| 🔴 **반드시 보고 + 직접 QA / 웬만하면 금지** | **저장·경로 검사·타임라인·영상 불러오기 처리**, **인코더/코덱 계약**, **콜아웃이 영상 시간에 맞춰 나오는 타이밍**, **xmeml/CapCut adapter와 PRIMARY 계산(`src/core`)** | 아니오. 잘못 건들면 데이터·출력이 조용히 깨질 수 있음 |

핵심 감각: **"눈에 보이는 문구·색"은 자유**, **"배치·해상도"는 물어보고**, **"저장·불러오기·코덱·parser·타이밍"은 위험**입니다.

## 2. "이걸 바꾸고 싶으면 이렇게 말하세요" 인텐트 표

| 하고 싶은 일 | 이렇게 말하면 됨 | 주의 |
|---|---|---|
| 버튼 이름 바꾸기 | "상단 툴바 버튼 글자를 ○○로 바꿔줘" | 🟢 버튼이 **하는 일**은 그대로, **글자만** |
| 도움말·안내 문구 수정 | "미리보기 도움말/드롭 안내 문구를 ○○로 고쳐줘" | 🟢 자유 |
| 색·폰트 톤 바꾸기 | "classic 공통 디자인 토큰의 액센트 색을 ○○로 바꿔줘" | 🟢 영역별 고정값은 `classic.css`에도 있으므로 함께 확인 |
| 자막(제목 콜아웃) 글씨 색·크기 | "영상 자막 글씨를 더 크게/다른 색으로 해줘" | 🟡 **"영상 시간 기반 타이밍은 바꾸지 마"**를 꼭 같이 (3번 참고) |
| 하단 레이아웃 배치 조정 | "레퍼런스 카드와 타임라인 배치를 ○○처럼 바꿔줘" | 🟡 `classic.css`부터 시작. 카드 크기·밀도는 `../CUSTOMIZING.ko.md`에 적힌 좁은 packing 지점도 확인하되 reference timing이나 `src/core`는 다시 쓰지 않음 |
| 출력 해상도·화면비 바꾸기 (예: 세로 4:5) | "render-spec의 width/height와 classic presentation 값으로 고정 ○:○ 포크를 만들어줘. `../CUSTOMIZING.ko.md`의 포크 가이드 따라서" | 🟡 새 canvas에 맞춰 배치하고 `CONFIG.panelHeight`도 확인. fps/bitrate는 별도 Export 계약이므로 함께 바꾸지 않음 |
| 새 기능 추가 | "○○ 기능을 **기존 동작은 그대로 두고** 옵션으로 추가해줘" | 🟡 기존 걸 **대체하지 말고 확장**하라고 명시 |
| 인코더/코덱 바꾸기 | (되도록 시키지 말기) "가능한지·위험한지부터 먼저 알려줘" | 🔴 BGRA 캡처와 원본 audio stream-copy는 고정 경계. 공식 기본값은 60fps이며 fps 변경은 별도 Export 계약 변경 |
| 저장이 안 돼요 / 경로 오류 | "고치지 말고 `docs/TROUBLESHOOTING.md`에 따라 `current-job/logs/app.log`의 최근 20–50개 이벤트를 보고 실패 단계를 알려줘" | 🔴 원인 확인이 먼저. transaction 오류는 마지막 한 줄만 보면 부족함 |

표에 없는 걸 시킬 땐, 0번 기본 주문을 붙이고 **"이거 위험한 부분이야, 아니야?"**를 먼저 물어보세요.

## 3. LLM이 순진하면 사고 치는 요청 (미리 막기)

말투가 자연스러워서 LLM이 덜컥 해버리기 쉬운데 실제로는 앱을 깨는 것들입니다. 이런 걸 원할 땐 **"○○ 계약 건드리지 말고"**를 꼭 덧붙이세요.

- **"자막을 스르륵 부드럽게 나타나게 해줘"** → 자막은 **영상 시간에 맞춰** 나와야 미리보기와 최종 출력 타이밍이 일치합니다. CSS wall-clock 애니메이션으로 바꾸면 출력이 어긋납니다. → "타이밍 로직은 그대로 두고 겉모양만".
- **"XML 읽는 걸 새로 깔끔하게 짜줘"** → `src/core`의 parser와 PRIMARY 계산은 안정 코어입니다. 다시 쓰지 마세요.
- **"내보내기 더 가볍게/작게 해줘"** (막연하게) → 막연한 요청이 위험합니다. 해상도를 바꾸고 싶으면 위 인텐트 표의 "화면비 포크"처럼 width/height와 배치를 명시하세요. 인코더/코덱 자체는 건드리지 마세요.
- **"저장 더 빠르게/간단하게 해줘"** → 저장은 일부러 안전하게(중간에 꺼져도 안 깨지게) 만든 겁니다. 건들지 마세요.

## 4. 바꾼 뒤 확인하는 법 (QA)

**1) 자동 검사 (LLM이 돌려줌)**

먼저 `npm.cmd run check`, 다음 `npm.cmd run smoke`를 실행해달라고 하세요. `CHECK_OK`와 `SMOKE_OK`는 현재 자동 회귀 검사 범위를 통과했다는 뜻입니다.

**2) 눈으로 확인 (당신이 함)**

자동 검사는 "색이 이상한가, 위치가 틀어졌나"를 판단하지 못합니다. 화면·자막처럼 보이는 변경은 앱을 직접 켜서 눈으로 봐야 합니다. LLM한테 "어디를 눌러 확인하면 되는지 순서로 알려줘"라고 하세요.

**3) 내보내기(Export) 변경**

`npm.cmd run smoke:export`는 임시 출력 경로를 자동 검사하고 결과를 삭제합니다. 마지막 시각 검수는 실제 앱의 `EXPORT H.264`로 내보낸 영상을 직접 확인하세요.

## 5. 문제 생겼을 때 되돌리기

- **아직 커밋 전이면**: 먼저 Git diff를 확인하고 **이번 요청에서 추가한 변경만** 되돌리라고 하세요. 같은 파일에 기존 작업이 섞여 있으면 사용자 확인 없이 파일 전체를 복구하거나 `reset`/`checkout`하지 말라고 명시하세요.
- **원인 모를 때**: 코드부터 고치지 말고 [`TROUBLESHOOTING.md`](TROUBLESHOOTING.md)를 따라 `current-job/logs/app.log`의 최근 20–50개 이벤트에서 operation/transaction 흐름을 먼저 확인하세요.
- **앱이 안 켜지거나 저장이 안 될 때**: 임의로 파일을 지우지 말고 상태부터 보고받으세요. 대부분 원본은 보존되도록 설계돼 있습니다.

## 한 줄 요약

**"바꾸기 전에 계약 읽어줘, 기존 작업 보존해줘, 위험하면 물어봐줘, 끝나면 check와 smoke 돌려줘"** — 이 습관이면 코드를 몰라도 훨씬 안전하게 시킬 수 있습니다.
